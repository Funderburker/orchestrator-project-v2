# Отчет: выбор платформы для мульти-агентной системы разработки

## Цель

Построить систему, где AI-агенты с разными ролями (backend, frontend, tester, deployer) автоматически выполняют задачи по разработке: пишут код, тестируют, деплоят. Задачи приходят через таск-менеджер или чат-бот.

## Что протестировали

### 1. LangGraph + свой код (Python)

**Что сделали:** полностью рабочий прототип — supervisor декомпозирует задачу, 4 агента работают в pipeline (backend+frontend параллельно, потом tester, потом deployer), каждый в своей git-ветке. Интеграции с Plane, Telegram-бот, уведомления, трекинг токенов.

**Плюсы:**
- Полный контроль над каждым шагом
- Работает с любым LLM-провайдером (Popus, Kie.ai)
- Pipeline с зависимостями (tester ждет backend)

**Минусы:**
- ~15 файлов кастомного кода (graph, state, worker, supervisor, tools, config, server, bot...)
- При реальном тесте агенты сожрали 800K токенов из-за зацикливания ReAct-агентов
- Много ручной работы: отладка, обёртки, фиксы
- Поддержка и развитие — всё на нас

**Вердикт:** работает, но слишком много своего кода.

---

### 2. n8n (визуальный автоматизатор)

**Что сделали:** развернули в Docker, создали workflow Telegram → Plane, настроили Telegram credential.

**Плюсы:**
- Визуальный редактор, 400+ интеграций
- Telegram, Slack, HTTP — готовые ноды

**Минусы:**
- Это автоматизатор, не агентная система. AI Agent нода — просто обёртка над LangChain
- Telegram Trigger требует HTTPS (нужен ngrok или SSL)
- Кастомная логика (file tools, git, workspace) — всё равно Code-нода с inline JS
- Отладка сложнее чем в коде
- Версионирование workflow — JSON вместо git diff

**Вердикт:** не для агентов. Подходит для простых автоматизаций, но ядро системы на нём не построишь.

---

### 3. OpenHands (open-source Devin)

**Что сделали:** развернули в Docker, подключили Kie.ai, попробовали дать задачу.

**Плюсы:**
- Полноценный AI-разработчик: файлы, терминал, git, дебаг
- Веб-интерфейс
- 70k+ звезд, активная разработка

**Минусы:**
- Это ОДИН агент, не мульти-агентная система
- Нет ролей (backend, tester, deployer) — всё делает один
- Нет pipeline и декомпозиции
- По сути Claude Code в браузере — не управляемая система

**Вердикт:** хорош для единичных задач, но не для оркестрации нескольких агентов.

---

### 4. Agent Orchestrator (Composio AO)

**Что сделали:** установили в WSL, настроили с Linear и GitHub, создали issue, запустили агента.

**Плюсы:**
- Готовая мульти-агентная система для кодинга
- Linear/GitHub/GitLab интеграции из коробки
- Dashboard, уведомления (Slack, Discord)
- Изолированные git worktree
- Автоматическая обработка CI-ошибок

**Минусы:**
- Требует GitHub/GitLab repo (нет локального режима)
- Агенты — Claude Code или aider, оба не работают с Kie.ai
- aider через прокси получает "Empty response" (streaming проблема)
- Windows-поддержка через WSL — нестабильно
- Завязан на подписку Claude Code или прямой Anthropic API

**Вердикт:** лучшая готовая система, но не работает с бюджетными LLM-провайдерами.

---

### 5. CrewAI

**Что сделали:** установили, подключили Kie.ai через наш прокси-сервер, запустили pipeline из 2 агентов (backend + tester).

**Плюсы:**
- Мульти-агенты с ролями — из коробки
- Sequential/parallel pipeline — одна строка
- Передача контекста между агентами (`context=[prev_task]`)
- Работает с любым LLM через LiteLLM (включая Kie.ai через прокси)
- Минимум кода: ~20 строк вместо ~500

**Минусы:**
- Tools (file, git, shell) всё равно нужно писать
- Интеграции (Telegram, Linear) — тоже кастом
- Документированный оверхед по токенам (~3x в бенчмарках, но на практике в тесте расход был минимальный)

**Вердикт:** лучший баланс "готовое vs кастомное". Оркестрация из коробки, допиливаем только tools и интеграции.

## Проблема с LLM-провайдерами

| Провайдер | Статус |
|---|---|
| Popus/KeyQuota | Закрылся |
| Kie.ai | Работает, но нестандартный API — Anthropic SDK и aider ломаются. Нужен прокси-сервер |
| Anthropic напрямую | Идеально, но дороже |

**Решение:** написали прокси-сервер (proxy/server.py), который принимает стандартные Anthropic и OpenAI запросы, форвардит в Kie.ai, фиксит ответ. Работает с CrewAI и нашим оркестратором.

---

## Итоговый выбор: CrewAI + прокси + кастомные tools

**Архитектура:**
```
Telegram/Linear → CrewAI (агенты с ролями) → Kie.ai (через прокси)
                        |
                  File/Git/Shell tools
                        |
                  Workspace (git repo)
```

**Почему CrewAI:**
- Убирает ~80% нашего boilerplate-кода
- Роли и pipeline работают из коробки
- Совместим с Kie.ai через прокси
- Дёшево: ~$0.25-1.00 за полноценную задачу

**Что остается написать:**
- File/git/shell tools (~100 строк)
- Прокси для Kie.ai (уже готов)
- Telegram-бот или веб-интерфейс
- Linear интеграция
